# Bible Accuracy Benchmark — v0.2

**Status:** current · **Generation:** 2026-07 · **Supersedes:** [v0.1](v0.1.md)

v0.2 keeps v0.1's structure — the same three deterministic tracks, the same
Overall Score formula, ~28 languages, and multiple translations — and improves
how faithfully the scorer detects quoted scripture. The goal of the changes is
singular: **be clear with models about how to quote, and reliably catch every
quote so we score quote accuracy fairly.**

> **Overall Score** = 100 × (0.50 × Direct Quotation + 0.25 × Scripture in Answers + 0.25 × Hallucination Resistance),
> each track averaged across the languages it covers.

For the full description of the three tracks and scoring, see [v0.1](v0.1.md);
only the differences are described below.

## Changes from v0.1

1. **A system prompt on every test (all tracks).** Each request now carries:
   > *"When you quote text from the Bible, put the quoted words inside double
   > quotation marks and give the reference (book chapter:verse). Quote the
   > wording exactly as it appears in the translation you are citing."*

   It is phrased **conditionally** ("when you quote"), so it does **not** force
   the implicit topical level (L2) to quote — it only standardizes the format
   when a model chooses to quote. This is fairer to the models (we tell them
   exactly what we want) and makes quote detection reliable.

2. **Simple-track prompt reconciled.** v0.1's direct-quote prompt told the model
   to respond "with no reference, no quotation marks" — which fought the new
   system prompt and the detector. v0.2 drops that; it asks only for the verse
   itself with no commentary, and lets the system prompt govern formatting. (The
   direct-quote scorer aligns against the known verse, so quotes/reference in the
   response are simply stripped before scoring.)

3. **Unquoted near-verbatim detection.** The topical and hallucination scorers
   now also catch scripture a model presents **without** quotation marks: each
   sentence outside the quoted spans is checked against the verse text, and a
   **near-verbatim match (≥ 90% character similarity)** is treated as a presented
   quote — accurate at ≥ 98%, a graded misquote between 90% and 98%. Reference
   attribution is applied the same way as for quoted spans (wrong verse →
   misattributed).

   Crucially, anything **below 90%** yields no verdict, so a paraphrase or
   allusion is **never** turned into a misquote — putting words in quotation
   marks remains the signal that earns full scrutiny. The backstop only ever adds
   a confident "this is verbatim verse X" detection; it never manufactures a
   misquote out of ordinary prose.

4. **Benchmark version comes from the codebase.** The eval runner no longer takes
   a `--run-version` flag; the version is the `BENCHMARK_VERSION` constant in the
   code (this file's generation). Bumping the code version defines a new, frozen
   generation; older runs keep their version.

## Why it matters

- **Fixes a false pass in Hallucination Resistance.** In v0.1, a model asked for
  a non-existent verse that recited a *real* verse in plain prose (no quotation
  marks) was scored "declined." v0.2 catches that unquoted substitution and
  correctly fails it.
- **Credits legitimate unquoted quotes** in topical answers that v0.1 missed.
- **Still protects paraphrase**, which the benchmark has always treated as
  legitimate and never penalized.

## Unchanged from v0.1
Tracks, weights, languages, translations, the character-level accuracy metric and
severity grades, the A×E topical scoring, the "declining scores zero" rules, the
deterministic-only principle, and the scope disclaimer (accuracy of quoted
scripture only, never theology).
